# 初始化服务器轨迹 SFT 统一工作目录

## 一、更新概述

在服务器 `/data3/llin` 下创建统一工作根目录 `trajory_sft`。从本轮开始，本项目新增的代码、配置、数据清单、训练输出和操作记录应统一组织在该目录下，避免继续分散到既有项目目录。

## 二、具体改动

创建目录：

```text
/data3/llin/trajory_sft
```

创建操作具有幂等性；目录已存在时不会覆盖或删除其中内容。

## 三、验证结果

- 路径：`/data3/llin/trajory_sft`
- 类型：目录
- 所有者：`root:root`
- 权限：`755`
- 所在文件系统：`/data3`
- 文件系统总容量：约 3.5 TiB
- 当前可用空间：约 1.5 TiB

## 四、后续目录约定

- 所有新的服务器端轨迹 SFT 工作默认以 `/data3/llin/trajory_sft` 为根目录。
- 不在该目录中保存明文密码、SSH 私钥、访问令牌等敏感信息。
- 大型数据、训练输出和 checkpoint 应进一步按用途划分子目录，避免文件直接堆放在根目录。

## 五、兼容性与影响

当前运行中的 `qwen3.5-9b-msswift-trajory-sft` 容器仍沿用源容器的旧挂载配置，新目录尚未 bind mount 到容器内。本轮没有重建或重启容器。后续如需在容器内直接使用该工作根目录，应先规划容器内目标路径，再安全重建容器挂载。

## 六、备份与回滚

当前目录为空时，可使用以下命令回滚；目录写入项目内容后不应直接删除：

```bash
rmdir /data3/llin/trajory_sft
```
