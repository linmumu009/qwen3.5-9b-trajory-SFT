# 切换至官方 A3 ms-swift v4.3.0 镜像并完成 NPU 验证

## 一、更新概述

核验服务器硬件与软件栈后，将 `qwen3.5-9b-msswift-trajory-sft` 从本地复制镜像切换为昇腾官方发布的 A3 版 ms-swift v4.3.0 镜像。新容器沿用原名称、工作目录、模型目录和 Ascend 驱动挂载，并通过 16 个 NPU 的可见性与最小张量算子验证。

本轮同时确认：服务器芯片标识 `ascend910_9391` 对应 Atlas 900 A3，而非 A2。A2 镜像虽然可以启动并识别设备，但其动态算子内核与本机芯片不匹配，不能作为训练环境。

## 二、环境核验与镜像选择

### 1. 宿主环境

- CPU 架构：`aarch64`
- Ascend 驱动版本：`26.0.rc1`
- CANN：`9.0.0`
- 芯片 SoC：`ascend910_9391`
- 硬件代际：Atlas 900 A3
- 可见 NPU 数量：16

### 2. 官方镜像

最终采用：

```text
quay.io/ascend/ms-swift:v4.3.0-A3-py311-CANN9.0.0-ubuntu22.04
manifest digest: sha256:0116ad4e0b2b440b3ff7353f24fca741a3173b1e9fcea595c99d358347f47952
```

该镜像与服务器的硬件代际、Python、CANN、操作系统和 ARM64 架构匹配。

核验过程中还拉取了以下 A2 镜像，但确认与本机 A3 动态算子内核不兼容，因此不使用：

```text
quay.io/ascend/ms-swift:v4.3.0-A2-py311-CANN9.0.0-ubuntu22.04
manifest digest: sha256:6a957416ab23dcae257b519f57c9694906906bc0d6236943077e478bd7d57c11
```

## 五、验证结果

### 1. 容器与挂载

- 容器状态：`running`
- 实际镜像：官方 A3 v4.3.0 镜像
- 容器工作目录：`/workspace/sft`
- 项目目录挂载来源：`/data3/llin/trajory_sft`
- 模型目录保持只读
- 缓存环境变量指向统一项目目录

### 2. NPU 验证

- `torch.npu.is_available()` 返回 `True`
- `torch.npu.device_count()` 返回 `16`
- 在 `npu:0` 创建张量并回传 CPU 成功，结果为 `[0, 1, 2, 3]`

### 3. A2 不兼容验证

A2 镜像可识别 16 个 NPU，但运行基础 `ones` 算子时报错：

```text
Parse dynamic kernel config fail
AclOpKernelInit failed
```

将环境变量覆盖为 `SOC_VERSION=ascend910_9391` 后仍失败，证明问题来自镜像内按 A2 编译的动态算子包，而不是单一运行时环境变量。

### 4. Qwen3.5 快路径依赖检查

官方 A3 镜像的基础 NPU 算子正常，但 ms-swift 的 Qwen3.5 NPU 快速线性注意力补丁尚不能完整导入，缺少：

```text
mindspeed.lite.ops.triton.chunk_delta_h
mindspeed.lite.ops.triton.chunk_scaled_dot_kkt
```

可正常导入：

```text
mindspeed.lite.ops.triton.chunk_o
mindspeed.lite.ops.triton.wy_fast
```

因此当前环境适合作为可复现的官方基础镜像，但尚未完成 Qwen3.5 轨迹 SFT 的模型级快路径闭环。

## 六、兼容性与影响

- 训练工作目录和模型路径不变，后续脚本无需因本次镜像切换调整路径。
- 原本运行中的本地复制容器已按要求删除。
- 原本地镜像 `qwen3.5-9b-msswift-trajory-sft:latest` 仍保留，可用于回滚。
- `qwen3.5-9b-msswift-trajory-sft-backup-v0.1.2` 备份容器仍保留，但它记录的是更早的旧工作目录挂载，不应直接作为当前目录结构的无检查回滚方案。
- A2 官方镜像只保留在本地镜像缓存中，不参与当前容器运行。

## 七、备份与回滚

如需回滚到本轮切换前的本地镜像，应先保存当前容器 inspect 配置，再删除当前容器，并使用以下本地镜像按本报告第三节的挂载参数重建：

```text
qwen3.5-9b-msswift-trajory-sft:latest
```

删除容器不会删除宿主机 `/data3/llin/trajory_sft` 中的数据，但回滚前仍应确认当前没有训练或数据写入任务。
