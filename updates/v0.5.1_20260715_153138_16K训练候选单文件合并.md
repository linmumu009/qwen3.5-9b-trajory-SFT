# v0.5.1 16K 训练候选单文件合并

- 版本：v0.5.1
- 日期：2026-07-15
- 更新类型：数据物化、完整性核验、文档
- 状态：已完成

## 一、更新概述

将 v0.5.0 已隔离的两类 16K 内 train 数据按原顺序合并为一个 ms-swift agent JSONL，便于后续统一抽样、复判和训练 smoke。源文件保持不变，held-out 与 16–32K 长轨迹没有进入合并文件。

合并组成：

| 来源 | 条数 | 状态 |
|---|---:|---|
| `train_strong_verified_16k.jsonl` | 27 | SQL 结果强验证 |
| `train_review_16k.jsonl` | 2,001 | 弱证据，仍需复判 |
| 合计 | 2,028 | 16K 内 train 候选 |

## 二、输出文件

物理机路径：

```text
/data3/llin/trajory_sft/data/screened/upstream_correct_screen_v1/train_candidates_16k_2028.jsonl
```

| 指标 | 结果 |
|---|---:|
| JSONL 行数 | 2,028 |
| 文件大小 | 65,224,332 B |
| SHA-256 | `45bf5fef2ae19c3417370a4adebdec7251ba520cfa0102be1f3352e5d2660aa7` |

合并顺序为强验证 27 条在前、待复判 2,001 条在后；没有去重、重写消息或改变原记录内容。

## 三、完整性验证

- 流式逐行 JSON 解析通过，`invalid_json=0`；
- 2,028 条全部包含非空 `messages` 和顶层 `tools`；
- 2,028 条最终消息角色均为 assistant；
- 共保留 39,799 条消息，其中 tool call 与 tool response 均为 12,677 条；
- 输出文件字节数等于两个源文件字节数之和；
- 两个源文件及 v0.5.0 的 held-out、长轨迹文件均未修改。

## 四、使用边界

- 该文件包含全部 2,028 条 16K 内 train 候选，但只有其中 27 条具有当前定义下的强验证证据；
- 其余 2,001 条仍需独立复判，不能把文件整体称为已经清洗完成的正式训练集；
- 文件名使用 `train_candidates`，特意不使用 `train_ready`；
- validation/test held-out 共 250 条和 16–32K 长轨迹 1,981 条未混入；
- 正式生产长度上限保持 16K。

## 五、回滚与复现

合并文件是两个不可变源文件的顺序拼接产物，不承担唯一数据源职责。需要回滚时只需停止引用该文件；需要复现时按相同顺序重新合并两个源文件并核对本报告中的 SHA-256。

## 六、下一步

1. 使用该文件做统一的数据抽样和复判入口，但保留强/弱证据标签来源；
2. 复判通过后另行生成 `train_ready_16k.jsonl`，不要覆盖本候选合集；
3. 首次训练 smoke 优先只取强验证子集，确认配置后再逐批加入复判通过样本。
