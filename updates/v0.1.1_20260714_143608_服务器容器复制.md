# 复制 Qwen3.5-9B 轨迹 SFT 训练容器

## 一、更新概述

以服务器现有容器 `qwen3.6-27b-msswift-sft` 为源，创建文件系统镜像快照，并按源容器的关键运行参数创建、启动新容器 `qwen3.5-9b-msswift-trajory-sft`。源容器未被删除或重启，复制完成后两个容器均保持运行。

## 二、操作前检查

- 源容器状态为 `running`，目标容器名称未被占用。
- 源容器内仅运行交互式 bash 和 TensorBoard，没有活跃训练进程。
- 源容器可写层约 537 MiB，根文件系统约 17.9 GiB。
- Docker 存储分区剩余约 2.1 TiB，空间充足。

## 三、具体改动

### 1. 创建镜像快照

从源容器生成以下镜像：

```text
qwen3.5-9b-msswift-trajory-sft:latest
```

镜像 ID：

```text
dcc31954b8bd78f204f22319d0012b20ad59c76db2f126dbee2650713d81c476
```

### 2. 创建并启动目标容器

目标容器沿用源容器的关键配置：

- Ascend runtime
- `host` 网络模式
- `host` IPC 模式
- privileged 模式
- `label=disable` 安全选项
- 交互式 TTY
- 工作目录 `/workspace/sft`
- 5 个 bind mount，其中模型目录为只读挂载
- 原有 Entrypoint、Cmd 和容器环境变量

目标容器 ID：

```text
7279ec5d5c77cecc5a4e8c0ae4a61d1342113116732989e56111f0adb733272f
```

## 四、验证结果

- 源容器 `qwen3.6-27b-msswift-sft`：`running`
- 新容器 `qwen3.5-9b-msswift-trajory-sft`：`running`
- 新容器镜像：`qwen3.5-9b-msswift-trajory-sft:latest`
- Runtime：`ascend`
- Network / IPC：`host` / `host`
- 挂载点数量：5
- Python：3.11.15
- `npu-smi info` 执行成功，Ascend 910 设备 0–15 均可见且健康
- 验证期间没有启动训练任务，也没有发现 NPU 计算进程

## 五、兼容性与影响

新容器与源容器共享相同的宿主机数据挂载，包括 `/workspace/sft` 对应的数据目录。任一容器对共享可写目录的修改都会被另一容器立即看到；后续并行训练时应使用不同输出子目录，避免覆盖 checkpoint、日志或评测结果。

## 六、备份与回滚

如需回滚本轮容器复制，可依次停止并删除目标容器，再删除目标镜像。源容器和其原始镜像不受影响。

```bash
docker stop qwen3.5-9b-msswift-trajory-sft
docker rm qwen3.5-9b-msswift-trajory-sft
docker rmi qwen3.5-9b-msswift-trajory-sft:latest
```
