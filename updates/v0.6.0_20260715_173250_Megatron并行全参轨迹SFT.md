# v0.6.0 Megatron 并行全参轨迹 SFT

- 版本：v0.6.0
- 日期：2026-07-15
- 更新类型：训练后端切换、并行拓扑、压力测试、正式训练
- 状态：正式训练运行中

## 一、更新概述

在 16 张 64 GiB Ascend A3 NPU 上，将 Qwen3.5-9B 的全参轨迹 SFT 从 Transformers/FSDP2 切换至镜像已经安装并面向昇腾适配的 Megatron-Core 0.16.0 + MindSpeed 0.16.0。先以 2,028 条候选中最长的 32 条完成两步真实前向、反向、梯度裁剪和 Adam 更新压力测试，再启动 150 步正式训练。

本轮训练的是语言模型主体；视觉塔和 aligner 按文本轨迹训练惯例冻结。输入继续采用完整的 `assistant/tool_call/tool_response` 轨迹，`loss_scale=default+ignore_empty_think`，工具响应保留为上下文但不作为监督目标。

## 二、FSDP2 故障定位与切换依据

此前的 FSDP2 尝试得到以下确定性结果：

| 方案 | 实际结果 | 结论 |
|---|---|---|
| FSDP2 full shard + activation checkpointing | 成功运行 8 步，第 9 步长样本反向 OOM | 峰值余量不足 |
| FSDP2 + activation CPU offload | 第 1 步成功，第 2 步反向 OOM | 优化器状态和持久显存仍是主要压力 |
| FSDP2 参数/梯度/优化器 + activation CPU offload | 梯度裁剪阶段触发 CPU DTensor all-reduce，HCCL 报无 CPU backend | 当前 NPU/HCCL 组合不适合作为生产路径 |

因此不再通过关闭梯度裁剪等方式绕过 FSDP 错误，而改用镜像内已经安装的 Megatron/MindSpeed 模型并行和分布式优化器路径。

## 三、并行与训练配置

世界规模为 16，正式拓扑为：

```text
TP 4 × PP 2 × DP 2 = 16
```

- Qwen3.5-9B 有 32 层，PP2 后每段 16 层；每段均保持 3 层 linear attention + 1 层 full attention 的相同节奏。
- hidden size 4,096、attention heads 16、KV heads 4 均可被 TP4 整除。
- TP4 与 sequence parallel 共同降低参数和长序列激活显存。
- PP2 进一步将 32 层切成两段。
- DP2 配合默认启用的 distributed optimizer（ZeRO-1）分摊优化器状态和梯度通信。
- micro batch 为 1，global batch 为 16，每个 DP 副本每步处理 8 个 micro-batch，PP2 理论流水线效率约 88.9%。

核心正式参数：

| 参数 | 值 |
|---|---:|
| 最大长度 | 16,384 |
| 精度 | BF16 |
| micro/global batch | 1 / 16 |
| 重计算 | full / uniform / 1 layer |
| 优化器 | Adam + distributed optimizer |
| 学习率 | 1e-6，cosine，min 1e-7 |
| warmup | 8 steps |
| 梯度裁剪 | 1.0 |
| weight decay | 0.1 |
| 正式步数 | 150 |
| 保存间隔 | 15 steps |
| 保存上限 | 10 checkpoints |

150 个优化器步共消费 2,400 个样本位置，约为 2,028 条候选的 1.18 个 epoch。

## 八、质量边界、风险与回滚

- 2,028 条数据由 27 条强验证和 2,001 条待复核候选组成；训练成功不代表剩余样本已经完成质量复判。
- 本轮不训练视觉塔和 aligner；“全参”指语言模型主体全参数，而不是对无图像输入的视觉模块做无监督更新。
- 若正式任务异常，保留运行目录中的 `train.log`、`logging.jsonl`、`npu_smi.log`、`resolved_config.txt` 和已完成检查点，不覆盖原始模型或数据。
- 断点续训应使用 MCore checkpoint，并保持 `TP4/PP2/DP2` 及优化器/RNG加载设置；不要直接把检查点目录当作 HF 模型使用，推理前需另行导出。
