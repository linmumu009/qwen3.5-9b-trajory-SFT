## 更新概述

以 `docs/trajectory_sft_process.md` 1.3 节中的真实强验证轨迹 `task_000201` 为唯一贯穿案例，新增一个无需外部依赖、可离线打开和打印的单页 HTML 报告。报告把轨迹 SFT 拆为可审计的九步过程，从原始 OpenAI-style 消息一直推导到 Qwen3.5-9B 的 masked next-token cross-entropy、反向传播和参数更新。

报告严格区分三类信息：原始轨迹中逐字保留的证据、1.3 节明确省略的长内容、根据真实 token 计数与仓库代码得到的数学推导。没有补造被省略的目录枚举和数据表列表命令。

## 关键口径

本报告使用以下已验证事实：

| 指标 | 值 |
|---|---:|
| 输入 token | 3,464 |
| 监督 token | 416 |
| 非监督 token | 3,048 |
| 监督密度 | 12.0092% |
| 工具调用 | 4 |
| SQL 工具结果 | 6,639 分钟 |

单样本目标在标准非忽略 token 均值口径下写为：

\[
\mathcal{L}_{sample}
=-\frac{1}{416}\sum_{j\in A}
\log P_\theta(x_j\mid x_{<j})
\]

其中 `A` 是 416 个 assistant 推理、工具调用和最终回答 token 的位置集合。其余 token 保留在 `input_ids` 中作为条件，但对应 `labels` 被置为 `-100`。

## 遗留问题

1. 1.3 节为控制公开材料长度省略了前两个探索命令及长输出；HTML 明确标为“公开节选省略”，没有伪造命令文本。
2. 报告中的 token 分区示意用于解释语义区域，不代表 tokenizer 的逐 token 切分；精确 token 以 `verify_trajectory_encoding.py` 输出为准。
3. 单条强验证样本只能说明流程和局部质量，不能证明 2,028 条候选全部正确，也不能替代 held-out 在线工具评测。
