# v0.7.1 轨迹 SFT 过程说明材料

- 版本：v0.7.1
- 日期：2026-07-16
- 更新类型：技术材料、原理说明、文档
- 状态：已完成

## 一、更新概述

新增 [`docs/trajectory_sft_process.md`](../docs/trajectory_sft_process.md)，用于严谨、精炼地讲清楚轨迹 SFT 从输入数据到参数更新的完整过程。材料以 OpenAI-style JSONL 为外部表示，不把基本原理绑定到 ms-swift；本项目的 Qwen3.5/ms-swift 映射单独放在实现章节。

## 二、材料覆盖范围

1. `messages`、`tools`、`assistant.tool_calls`、`role=tool` 和 `tool_call_id` 的完整样例。
2. `reasoning_content` 与 OpenAI 基础字段的边界说明。
3. 普通 SFT 与轨迹 SFT 的区别。
4. chat template、tokenization 和 decoder-only causal attention。
5. system/user/tools/observation/assistant/padding 的 loss mask 定义。
6. logits、softmax、next-token probability、masked cross-entropy 和 batch loss 的逐步推导。
7. teacher forcing、反向传播、AdamW、梯度累积和 LoRA 更新。
8. “observation 留在输入、labels 设为 IGNORE_INDEX”的严格解释。
9. 离线训练重放与在线推理工具闭环的差异。
10. 格式、正确性、去重、防泄漏、长序列、安全和评测闭环。
11. 本项目实际 mask 规则和可直接用于汇报的五句话总结。

## 三、依据与边界

- OpenAI-style 格式依据 OpenAI 官方 SFT 和 Function Calling 文档核对。
- Transformer 因果序列建模与 LoRA 分别引用原始论文。
- 本项目实现依据 `prepare_trajectory_sft.py`、`verify_trajectory_encoding.py` 和既有真实 Qwen3.5 模板验收。
- `reasoning_content` 明确标为项目扩展字段，不描述为 OpenAI Chat Completions 基础标准字段。
- 材料不声称格式严格候选均已完成正确性复判。

## 四、验证结果

- Markdown 标题、代码块、表格、公式分隔符与相对链接完成静态检查。
- 公式统一采用“位置 \(t\) 的 logits 预测 \(x_{t+1}\)”定义，label shift 前后一致。
- causal attention mask、padding mask 和 loss mask 分开描述，避免概念混淆。
- 项目 mask 结论与既有验证一致：assistant 推理/tool call/最终回答参与 loss，tool response 不参与 loss但保留上下文。
- 项目测试：`27 passed`；公开材料敏感信息扫描未发现服务器地址、登录信息或本地数据路径。

## 五、兼容性与影响

本轮只新增说明材料和版本记录，不修改数据、训练脚本、模型、容器、检查点或服务器产物。文档主线是框架无关原理，可用于技术汇报；实现章节对应当前 Qwen3.5-9B/ms-swift 工程。
